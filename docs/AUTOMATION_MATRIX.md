# 03_AUTOMATION_MATRIX - кому доверяем какой процесс

Требование заказчика: явно видеть, какой уровень автономии у каждого шага и почему.
Уровни: **script** (детерминированный код), **loop** (script по расписанию/с retry),
**LLM offline** (разовая помощь модели вне рантайма), **human** (только человек).

| # | Процесс | Уровень | Почему именно так | Когда можно поднять уровень |
|---|---------|---------|-------------------|------------------------------|
| 1 | Сбор вакансий из API/RSS | loop (cron daily) | Детерминированно, дешево, источники стабильны | Уже максимальный |
| 2 | Нормализация полей | script | Правила фиксированы, покрыты тестами на fixtures | - |
| 3 | Парсинг зарплат | script | Regex + таблица кейсов; ошибки видны в salary_raw | LLM offline: разово прогнать 100 нераспознанных salary_raw и расширить regex |
| 4 | Дедупликация | script | Hash-функция, тестируется | - |
| 5 | Hard-фильтры | script | Бинарные правила из config | - |
| 6 | Скоринг | script (rules + локальные embeddings) | Воспроизводимо, бесплатно, объяснимо (reasons/risks) | LLM offline: калибровка весов по 100-200 вручную размеченным вакансиям (scripts/calibrate.py, v2) |
| 7 | Retention/VACUUM БД | loop | Механическая гигиена | - |
| 8 | Дайджест в Telegram | loop | Шаблонная генерация текста | - |
| 9 | Мониторинг здоровья пайплайна | loop + human | Actions падает -> e-mail; ошибки источников -> в дайджест; чинит человек (с LLM в чате) | Автопочинка парсеров - осознанно НЕ делаем: источник поменял формат = нужно решение человека |
| 10 | Решение "откликаться или нет" | **human** | Score - это сортировка внимания, не решение. Цена ложного отклика/пропуска высокая | Никогда не выше. Принципиальная позиция, хорошо звучит на собеседовании |
| 11 | Tailored CV / cover letter | LLM offline + human | LLM готовит черновик под top-вакансию, человек проверяет честность формулировок | Не автоматизируется: отправка = human |
| 12 | Отправка откликов | **human** | ToS площадок, репутация, антиспам | Никогда |
| 13 | Правка статусов (applied/ignored) | human | Ручная пометка в CSV/sqlite (любым sqlite-браузером) | v2: маленькая Streamlit-страница, если надоест руками |
| 14 | Обновление словарей/весов config | human (по сигналам из дайджестов) | Дрейф качества виден человеку; изменения = git commit = история | LLM offline помогает предлагать правки |

Итог для собеседования одной фразой: детерминированный пайплайн в цикле делает 100%
ежедневной механики; LLM подключается только офлайн для калибровки и подготовки материалов;
все необратимые действия (отклик, отправка, суждение о fit) остаются за человеком by design.
